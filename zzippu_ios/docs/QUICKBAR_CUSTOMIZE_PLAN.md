# 홈 빠른기록 버튼 커스터마이즈 — 기획/디자인 스펙

> 대상: 홈 상단 빠른기록 바(현재 `BigActionGrid`, 하드코딩 8버튼, 1줄 가로 스크롤).
> 목표: 사용자가 **순서 변경 · 숨기기 · 다시 추가**를 할 수 있게. 기기 단위 로컬 저장.
> 개발: 후속 Sonnet가 이 스펙으로 구현. 본 문서는 코드 미포함(구현 가능한 구체값까지 확정).

## 0. 핵심 결정 요약(TL;DR)

| 항목 | 결정 |
|---|---|
| 편집 방식 | **(A) 전용 "빠른기록 편집" 시트** (드래그 재정렬 리스트 + 표시/숨김 2섹션). 인플레이스 지글모드(B)는 가로 드래그 리스크로 기각 |
| 진입점 | **바 맨 끝 고정 "편집" 칩**(연필 아이콘) + 보조로 바 롱프레스, 설정 경로 없음 |
| 저장 | `UserDefaults` 단일 키 `quickbar.visibleKinds` = 표시 kind rawValue **배열 JSON**(순서=배열 순서, 숨김=배열에 없음) |

| 마이그레이션 | 앱 업데이트로 신규 kind 추가 시 저장값에 없는 kind는 **표시 목록 끝에 append** |
| 재사용 컴포넌트 | `DSBottomSheet`(편집 시트) · `DSButton`(하단 완료/기본값 복원) · `DSChip`/DS 리스트로우 · `DSEmptyState`(숨김 섹션 빈 상태) |
| 세션 안전성 | 수면 버튼 숨겨도 활성 세션 로직 무관(세션은 저장소 상태 기반). 단 **활성 세션 중엔 수면 버튼 자동 노출**(엣지 §5.2) |

## 1. 데이터 모델

### 1.1 `QuickAction` — 버튼 1개의 추상화

현재 `BigActionButton(emoji:label:kind:){onAction(...)}`가 뷰 안에 8번 하드코딩돼 있다.
이를 값 타입 `QuickAction`으로 추출하고, 바는 **정렬된 `[QuickAction]`** 을 `ForEach`로 렌더한다.

```
struct QuickAction (개념 정의 — 코드 아님)
  ├─ kind:   QuickButtonKind      // 안정 식별자(저장 키). Theme.swift 기존 enum 재사용
  ├─ emoji:  String               // "🍼" 등
  ├─ label:  String               // "분유" (수면은 예외: 세션상태로 라벨 토글 — §5.2)
  ├─ action: HomeAction           // 탭 시 라우팅될 액션(HomeView.handleAction 기존 스위치)
  └─ isSessionToggle: Bool        // 수면만 true(활성세션 시 라벨/색 토글)
```

- `QuickButtonKind`(Theme.swift, `public enum`)를 **저장용 안정 ID**로 승격.
  → 저장을 위해 `String` rawValue가 필요하므로 **`: String` raw-representable + `CaseIterable`로 확장**(값은 case명과 동일: `"formula"` 등). 색 매핑 `theme.color.quickButton(kind)`는 그대로 사용.
- `kind ↔ HomeAction`은 1:1 이므로 매핑 함수 하나로 충분(둘 다 formula/breast/pee/poo/sleep/play/supplement/medicine 8종 동일).

### 1.2 카탈로그(전체 정의) vs 사용자 설정

- **카탈로그**: 앱이 아는 **모든** 버튼의 순서·이모지·라벨. 코드 상수(단일 소스). = 현재 8개 정의 그대로 옮김.
- **사용자 설정**: "표시할 kind를 어떤 순서로". 카탈로그의 부분집합 + 순서.
- 렌더 = `설정에 담긴 kind 순서대로, 카탈로그에서 QuickAction 조회`.

## 2. 저장(영속)

### 2.1 스키마 — 표시 배열 단일 키

```
UserDefaults.standard
  key:   "quickbar.visibleKinds"
  value: JSON String 배열, 예) ["pee","poo","formula","sleep"]
         • 배열 = "표시" 목록
         • 순서 = 화면 순서
         • 카탈로그에 있으나 배열에 없는 kind = "숨김"
```

- **왜 "숨김 배열"이 아니라 "표시 배열"인가**: 순서와 표시여부를 **한 배열**로 표현 → 상태 원자성↑(순서/숨김 불일치 불가). 숨김은 "표시배열 여집합"으로 파생.
- 접근 패턴은 기존 `OfflineToggle`과 동일하게 **enum 네임스페이스 + computed property**(예: `QuickBarSettings.visibleKinds` get/set). @AppStorage 미사용(코드베이스 관례 준수, JSON 배열이라 부적합).
- 잘못된 JSON/디코드 실패 → **기본값으로 폴백**(방어적).

### 2.2 기기 단위로 충분한가 — 판단: **충분(기기 단위 채택)**

- 근거: 빠른기록 바는 **입력 UX 선호**일 뿐 데이터가 아님. 계정/아기별로 다를 이유가 약하고, 서버 스키마·동기화 비용(콜드스타트 방어 정책상 서버 왕복 최소화)만 늘린다.
- local-first 원칙(계정만 서버) 및 `UserDefaults` 관례와 정합. 멀티기기 편차는 감수(추후 필요 시 계정 프로필로 승격 여지만 남김 — §7 리스크).
- 아기 전환 시에도 바 구성 유지(양육 흐름 동일).

### 2.3 기본값 & 마이그레이션

- **기본값**(키 미설정 시): 카탈로그 전체 순서 그대로
  `["formula","breast","pee","poo","sleep","play","supplement","medicine"]`.
- **신규 버튼 추가 마이그레이션**: 로드 시 `카탈로그에 있으나 저장 배열에 없는 kind`를 찾아 **표시 목록 끝에 append**(사용자가 의도적으로 숨긴 게 아니라 "그 버전엔 없던 것"이므로 노출이 안전). append 후 즉시 저장(1회성 정규화).
  - 반대로 **카탈로그에서 사라진 kind**(폐기)는 로드 시 조용히 드롭.

## 3. 편집 인터랙션 — 대안 비교 & 택1

### 3.1 대안 비교

| | (A) 전용 편집 시트 ✅권고 | (B) 인플레이스 지글모드 |
|---|---|---|
| 구현 견고성 | **높음** — `List` + `.onMove`/`EditMode`가 표준. 세로 드래그 = SwiftUI 기본 지원 | **낮음** — 바가 **가로** 스크롤이라 가로 드래그 재정렬은 커스텀 제스처 필요. 기존 `QuickPressButtonStyle`가 스크롤 제스처와 충돌 회피용으로 특별 설계된 만큼(주석 참조), 지글+드래그 얹으면 회귀 위험 큼 |
| 발견성 | 중(명시적 "편집" 진입점 필요) | 중(롱프레스 관례에 의존) |
| 친숙함(HIG) | 높음 — iOS "편집" 리스트 패턴, VoiceOver 재정렬 무료 | 높음(홈화면식)이나 육아앱엔 과함 |
| 접근성 | **강함** — `EditMode`에서 VoiceOver 재정렬 accessibilityAction 기본 제공 | 약함 — 흔들림·X 커스텀 접근성 수작업 |
| 빈/숨김 관리 | 섹션으로 자연스러움 | 별도 트레이 UI 추가 필요 |

### 3.2 권고: **(A) 전용 "빠른기록 편집" 시트**

- SwiftUI에서 가장 견고하고, 가로 스크롤 바의 드래그 난제를 회피. 접근성·다크·DS 정합이 전부 표준 경로로 해결.

### 3.3 진입점 확정

1. **주 진입점 — 바 맨 끝 고정 "편집" 칩**: 가로 스크롤 바의 **마지막 항목**으로 `연필(pencil) 아이콘 + "편집"` 칩을 항상 노출(스크롤 대상 아닌 sticky 고려하되, MVP는 스크롤 내 마지막 셀로 단순화). 탭 → 편집 시트 오픈.
2. **보조 — 바 영역 롱프레스**(0.5s): 어디를 눌러도 편집 시트 오픈(발견성 보강).
3. 설정(SettingsView) 경로는 **두지 않음**(빠른기록은 홈 맥락에서 편집하는 게 자연스럽고, 설정에 항목 추가 시 IA만 무거워짐). 필요 시 추후.

## 4. 시각 디자인 (편집 시트)

### 4.1 시트 골격 — `DSBottomSheet`

```
options: .init(title: "빠른기록 편집", detents: [.large])   // 재정렬 여유 위해 large 고정
```

편집 시트 = **표시 섹션 + 숨김 섹션 2개**를 담은 리스트. `EditMode.active`로 진입(드래그 핸들·삭제 컨트롤 상시 노출).

```
┌──────────────────────────────────────┐
│  빠른기록 편집                    [완료] │   ← DSBottomSheet 헤더
├──────────────────────────────────────┤
│  표시 중                                │   ← 섹션 헤더(DSSectionHeader)
│  ⊟  🍼 분유                        ≡    │   ← ⊟=숨김(빼기), ≡=드래그 핸들
│  ⊟  💧 소변                        ≡    │
│  ⊟  💩 대변                        ≡    │
│  ⊟  😴 수면                        ≡    │   (표시 1개만 남으면 그 행의 ⊟ 비활성)
│                                        │
│  숨김                                   │   ← 섹션 헤더
│  ⊕  🤱 모유                             │   ← ⊕=추가(표시로 이동, 표시 맨 끝에)
│  ⊕  🎈 터미타임                          │
│  ⊕  🧴 영양제                            │
│  ⊕  💊 약                               │
│  (숨김 없음 시) → DSEmptyState "숨긴 버튼이 없어요" │
├──────────────────────────────────────┤
│           [ 기본값으로 복원 ]            │   ← DSButton(.tertiary/.ghost)
└──────────────────────────────────────┘
```

### 4.2 컨트롤 규칙

- **재정렬**: `표시 중` 섹션 내 `.onMove`. 드래그 핸들(≡, `line.3.horizontal`)은 EditMode에서 자동.
- **숨기기**: `표시 중` 행 좌측 빨간 ⊟(`minus.circle.fill`) 탭 → 숨김 섹션 끝으로 이동. (스와이프-삭제 대신 명시 버튼: 실수 방지 + VoiceOver 명확)
- **추가**: `숨김` 행 좌측 초록 ⊕(`plus.circle.fill`) 탭 → 표시 섹션 **맨 끝**으로 이동.
- **미리보기(선택)**: 시트 최상단에 실제 바를 축소 렌더해 변경 즉시 반영(MVP 제외 가능, §6 S6에 옵션으로).
- **저장 시점**: 각 조작마다 즉시 in-memory 반영 + `[완료]` 또는 시트 dismiss 시 `UserDefaults` 커밋. (조작마다 커밋해도 무방 — I/O 저렴)

### 4.3 DS 정합·다크·접근성

- 색/타이포: 모든 색은 `theme.color.*` 토큰, 라벨 `theme.typography.*`. 하드코딩 색 금지 → 다크 자동 대응.
- 버튼 미리보기 셀은 기존 `theme.color.quickButton(kind)` 팔레트 재사용(일관성).
- **터치 타깃 ≥44pt**: ⊟/⊕/핸들 모두 44 최소.
- **VoiceOver**:
  - 재정렬은 EditMode 기본 accessibilityAction("위로 이동/아래로 이동") 사용.
  - ⊟ 라벨 "분유 숨기기", ⊕ 라벨 "모유 추가". 행 값에 현재 순서 위치 안내("표시 3개 중 1번째").
- **다이나믹 타입**: 라벨 줄바꿈/스케일 허용(바 버튼은 기존 `minimumScaleFactor(0.8)` 유지).

## 5. 엣지 케이스

### 5.1 모두 숨김 방지 — **최소 1개 강제**
- 표시 개수 == 1이면 그 행의 ⊟ 비활성(회색) + 탭 시 토스트 "최소 1개는 남겨두세요"(info).
- 저장 계층에서도 방어: 커밋 시 빈 배열이면 기본값으로 폴백.

### 5.2 활성 세션(수면) 안전성
- 세션 시작/종료 로직은 **저장소의 활성 세션 상태**(`vm.activeSleepSession`) 기반 — 버튼 표시 여부와 **완전 독립**. 수면 버튼 숨겨도 진행 중 세션은 정상 유지되고 다른 경로(타임라인 행 편집 등)로 종료 가능.
- **단, UX 함정 방지**: 활성 수면 세션이 있는데 수면 버튼이 숨김이면 종료 진입점이 사라짐.
  → 규칙: **활성 세션이 있는 동안 수면 버튼은 숨김 설정과 무관하게 바에 자동 노출**(세션 종료 후 원래 설정 복귀). 렌더 단계에서 `hasActiveSleep`이면 `sleep`을 표시목록에 강제 포함(중복 없이). 저장값은 변경하지 않음.

### 5.3 미구현 기능(영양제·약)도 커스터마이즈 대상 — **포함**
- 이미 바에 노출돼 "곧 지원" 토스트를 내는 기존 동작 유지. 커스터마이즈에서 숨기면 아예 사라져 오히려 깔끔(안 쓰는 사용자 니즈 부합). 별도 분기 불필요.

### 5.4 기본값 복원
- `[기본값으로 복원]` → 저장 배열을 기본값(§2.3)으로 리셋 + 확인 다이얼로그(`confirmationDialog`, "빠른기록 버튼을 기본으로 되돌릴까요?").

### 5.5 과거 날짜 포커스 모드(PastFocusView)
- 동일한 정렬된 `[QuickAction]`을 공유(바 구성 한 벌). 단 편집 진입점(편집 칩/롱프레스)은 **오늘 뷰에만** 노출 권고(과거 입력 중 편집 진입은 혼란). 과거 모드에선 편집 칩 숨김.

## 6. 개발 태스크 (S1 → S7)

- **S1 · 데이터 모델**: `QuickButtonKind`를 `String` raw + `CaseIterable`로 확장. `QuickAction` 값 타입 + 카탈로그 상수(8개) + `kind↔HomeAction` 매핑 정의.
- **S2 · 저장소**: `QuickBarSettings` enum 네임스페이스(`OfflineToggle` 패턴) — `visibleKinds: [QuickButtonKind]` get/set(UserDefaults JSON), 기본값·디코드 폴백·마이그레이션(append 신규/드롭 폐기).
- **S3 · 바 데이터 구동**: `BigActionGrid`를 하드코딩 → `ForEach(orderedActions)`로 전환. `hasActiveSleep` 시 수면 강제 포함(§5.2). 색/라벨은 기존 로직 유지. **탭 라우팅(handleAction) 무변경** 검증.
- **S4 · 편집 진입점**: 바 끝 "편집" 칩 + 바 롱프레스 → 편집 시트 상태 오픈(오늘 뷰 한정, §5.5).
- **S5 · 편집 UI**: `DSBottomSheet` + EditMode 리스트(표시/숨김 2섹션, ⊟/⊕/드래그, DSEmptyState, 기본값 복원 DSButton). VoiceOver·44pt 반영.
- **S6 · 영속 연결**: 편집 조작 ↔ `QuickBarSettings` 커밋, 시트 닫으면 바 즉시 반영. (옵션) 시트 상단 라이브 미리보기.
- **S7 · 엣지 & QA**: 최소1개 강제·모두숨김 폴백·활성세션 노출·기본값복원·마이그레이션(신규 kind append)·다크/다이나믹타입/VoiceOver 수동 점검.

## 7. 리스크
